# Contributing to Still Systems

Thank you for your interest in contributing! Still Systems is a workshop focused on building stable, foundational developer tools. We value quality, clarity, and long-term maintainability.

## Our Philosophy
- **Stability First**: We prioritize bug fixes and structural improvements over new features.
- **Clarity**: Documentation is as important as code.
- **Accessibility**: We strive to make our tools easy to install and use for everyone.

## How to Contribute

### Reporting Bugs
- Check the issues tab in the relevant repository to see if the bug has already been reported.
- If not, open a new issue with a clear title and a detailed description of the problem, including steps to reproduce.

### Suggesting Enhancements
- Open an issue to discuss the enhancement before starting any work.
- Focus on how the change aligns with our mission of "structural clarity and long-term stability."

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your code follows the existing style of the project.
3. Update documentation to reflect any changes.
4. Open a PR with a clear description of what was changed and why.

## Development Standards
- Keep PRs focused. One feature or fix per PR.
- Ensure all tests pass.
- Write clear, concise commit messages.

Thank you for helping us build better tools!
