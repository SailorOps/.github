# Support for Still Systems

If you need help with a Still Systems tool, please follow the steps below.

## 1. Check the Documentation
Before opening an issue, please check the official documentation at [stillsystems.github.io](https://stillsystems.github.io) or the `README.md` of the specific repository.

## 2. Search Existing Issues
Check the **Issues** tab of the relevant repository to see if your question has already been answered or if the bug has been reported.

## 3. Open an Issue
If you can't find an answer, please open a new issue in the repository related to your problem. Provide as much context as possible:
- What you were trying to do.
- What happened (including error messages).
- Your environment (OS, version of the tool, etc.).

## 4. Security Issues
If you believe you have found a security vulnerability, please do **not** open a public issue. Refer to our [Security Policy](./SECURITY.md) for reporting instructions.

---

*Still Systems is a volunteer-led workshop. We appreciate your patience as we work through issues.*
